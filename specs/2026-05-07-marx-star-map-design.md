## 1. Context · 这次 design 的来由

上轮 brainstorming（2026-05-06，PRD v0.2）定到路径 A「学者作品集型」+ 关系星图主视图 + 8-12 周 MVP，然后暂停去做用户访谈校验。2026-05-07 访谈完成，3-5 位目标用户给了 8 条反馈。其中：

- **2 个重大调整**：(a) 主视图从单一关系图改为 B+2 双主视图（地理图画中画副）；(b) 在核心概念详情卡里加「后来者发展」旁注栏
- **3 个细化方向**：节点 bio 必须事件式简明 / 框架感必须在主视图就一眼看出 / V1 工具型为主 + V2 加故事模式入口给路人

本 design doc 落实调整后的产品 spec，为下一步 writing-plans 提供完整输入。

## 3. 节点系统

### 3.1 5 类节点 spec

| 类型 | 颜色 | 数量 | 必填字段 (V1) | 选填字段 (V1 录入，V2+ 渲染用) |
|---|---|---|---|---|
| **人** | 紫 | ~50 | `id` / `name_zh` / `name_orig` / `birth_year` / `death_year` / `main_location_lat_lng` / `bio_event_style`(≤5 行) / `citation_urls`[] | `aliases`[] / `portrait_url` / `relation_to_marx` |
| **著作** | 蓝 | ~20 | `id` / `name_zh` / `name_orig` / `pub_year` / `author_id` / `writing_period` / `summary`(≤3 行) / `citation_urls`[] | `version` / `writing_location_lat_lng` ⭐ |
| **事件** | 橙 | ~30 | `id` / `name_zh` / `start_date` / `end_date` / `location_lat_lng` / `description_event_style`(≤3 行) / `participants_ids`[] | `triggered_works_ids`[] |
| **概念** | 绿 | ~15 | `id` / `name_zh` / `name_orig` / `proposed_year` / `proposed_work_id` / `definition_plain`(≤3 行) / `citation_urls`[] / `successor_notes`[] (核心概念) | `related_concepts_ids`[] / `evolution_path` / `first_proposed_location_lat_lng` ⭐ |
| **地点** | 灰 | ~6 | `id` / `name_zh` / `name_orig` / `lat_lng` / `marx_activity_description` | `related_persons_ids`[] / `related_events_ids`[] |

**总计 ~121 节点 + 余量 29 ≈ 150**。

### 3.2 节点 bio 事件式 + 简明强制规则

命中访谈反馈 1（学术语言晦涩冗长）：节点 bio / description / summary 字段**禁止学术包装**，必须采用「x 年 x 月 X 做了什么」事件流写法，每行 ≤ 30 字，总长度 ≤ 5 行（人）/ ≤ 3 行（著作 / 事件 / 概念 / 地点）。

**反例**（学术包装，禁止）：
> 弗里德里希·恩格斯（1820-1895）是 19 世纪伟大的无产阶级革命家、思想家，他与马克思的合作是马克思主义形成发展中的关键因素……

**正例**（事件式简明，必须）：
> 1842年9月 - 在《莱茵报》编辑部第一次见 Marx，关系冷淡
> 1844年8月 - 巴黎再见，10 天彻夜长谈，思想合流
> 1845年 - 合写《神圣家族》《德意志意识形态》
> 1848年 - 合写《共产党宣言》
> 1851-1869年 - 曼彻斯特工厂工作，养活伦敦的 Marx 一家

### 3.3 两图渲染规则

| Tab | V1 默认渲染 | V2+ 可渐进打开 |
|---|---|---|
| **关系图** | 5 类全渲染 (~150 节点) | — |
| **地理图** | 仅紫+橙+灰 三类 (~86 节点) | 「显示著作」checkbox（V2 打开）/「显示概念」checkbox（V3+ 打开） |

**保留这条扩展路径的理由**：用户洞察「历史/地理/政治不分家」——思想形成有强地理上下文。著作的「主要写作地」、概念的「首次提出地」字段在 V1 数据采集时就一并录入，等 V2+ 通过筛选 checkbox 渐进打开。V1 暂不默认渲染是为了控制信息密度风险，**不预设否定**。

## 5. 后来者旁注 panel · 访谈最大变化点

### 5.1 触发位置

**仅在核心概念节点的详情卡里**出现「后来者发展」栏。其他节点（人 / 著作 / 事件 / 地点）的详情卡**不带**这一栏。

### 5.2 V1 必做范围

**12 个核心概念**：
1. 异化 (Verfremdung / Entfremdung)
2. 剩余价值 (Mehrwert)
3. 历史唯物主义 (Historischer Materialismus)
4. 阶级斗争 (Klassenkampf)
5. 商品拜物教 (Warenfetischismus)
6. 意识形态 (Ideologie)
7. 生产方式 (Produktionsweise)
8. 经济基础 / 上层建筑 (Basis / Überbau)
9. 共产主义 (Kommunismus)
10. 革命 (Revolution)
11. 劳动价值论 (Arbeitswerttheorie)
12. 国家 (Staat)

**10 位后来者池**：
卢卡奇 / 葛兰西 / 阿尔都塞 / 哈贝马斯 / 卢森堡 / 列宁 / 马尔库塞 / 哈维 / 詹姆逊 / 巴利巴尔。

每个核心概念挑 3-5 位后来者做旁注（不是 10 位全列）。

### 5.3 旁注内容结构

每条旁注约 300-500 字 + 3-5 个引用源链接。结构示范（以「异化」为例）：

```
[详情卡 · 异化]
─────────────────────────────────
中文：异化      原文：Verfremdung / Entfremdung
提出：1844 年《1844 年经济学哲学手稿》

—— 白话定义 ——
人在资本主义生产中，跟"劳动产品 / 劳动过程 / 自己的本质 / 他人"
四重疏离。例：工人造的车不属于自己。

—— Marx 自己怎么用 ——
1844 手稿首次提出 → 1857-58 大纲改用"对象化" → 1867 资本论隐含
在"商品拜物教"里。

—— 后来者怎么发展 ——  ⭐ 访谈反馈 8 新加
• 卢卡奇 1923《历史与阶级意识》：扩展为"物化"（Verdinglichung），
  推到资本主义意识结构层面
• 阿尔都塞 1965《保卫马克思》：批为"青年 Marx 的人本主义残留"，
  主张用"意识形态国家机器"取代
• 哈贝马斯 1973《合法化危机》：转译为"生活世界殖民化"，
  接入沟通行动理论
• 马尔库塞 1964《单向度的人》：拓展到消费社会的异化新形态

—— 引用源 ——
• MIA《1844 手稿》中/英/德文全文
• SEP "Marx" 词条第 4 节 "Alienation"
• SEP "Lukács" / "Habermas" / "Althusser" 各自词条
```

> **信息密度提示**（v0.3 用户反馈补充）：本节示例展示的是「白话定义 + Marx 自己怎么用 + 后来者怎么发展 + 引用源」四段式完整内容，密度较高。实际渲染时可能需要做二次筛选（如默认折叠后来者发展 / 引用源 panel，用户点击展开），或通过分层分类的前端布局（缩进 / 配色 / 字号 / 折叠展开交互）降低密度感，让用户觉得舒服而不是密。这部分视觉细节延后到前端实施阶段调 `frontend-design` / `ui-ux-pro-max` 等 UI/UX 专门 skill 时讨论。

### 5.4 数据来源

- **主要**：SEP（Stanford Encyclopedia of Philosophy）各后来者词条（卢卡奇 / 哈贝马斯 / 阿尔都塞 / 葛兰西 都有自己的 SEP 词条）
- **二手**：后来者代表作（每位 1-2 本核心著作，如卢卡奇《历史与阶级意识》/ 阿尔都塞《保卫马克思》/ 哈贝马斯《合法化危机》）
- **中文校对**：《剑桥马克思主义指南》中译本 + CNKI 中文学术论文

### 5.5 与原 PRD「不做后续 marxist 思想史扩展」的关系

不冲突。原 PRD 第 5 节排除的是「把后来者做成主图节点」（方案 1）。本 design **V1 范围内**做的是「在概念详情卡里加旁注」（方案 2）—— V1 范围内后来者不出现在主图任一 tab，只在核心概念节点详情卡的下半部分作为参考资料。

**远期不预设否定**：V2+ 根据用户反馈，方案 2 仍可能演化为「后来者混入主图」（方案 1）/「独立 Marx 之后视图」（方案 3）/「方案 2 + 方案 3 双视图切换」。详见 § 9.3。PRD v0.3 第 5 节做对应澄清。

## 7. 首屏体验设计 · 命中访谈反馈的 4 条策略

| 策略 | 命中反馈 | 实现方式 |
|---|---|---|
| **1. 首屏即骨架** | 反馈 3 工具型 | 默认进关系图主视图 + 地理图画中画副，**无欢迎页 / 无引导动画**。学者打开就能用 |
| **2. 框架感一眼看出** | 反馈 4 框架清晰 | (a) 5 色节点编码，主视图一眼看出 Marx 思想生态构成；(b) 时间轴 4 阶段标记，拖动时看见思想发展阶段而非冰冷年份 |
| **3. 跨图搜索栏** | 反馈 3+4 便于查找 | 搜索词在主+副两图同时即时高亮，不必切换 |
| **4. 节点 bio 事件式** | 反馈 1 简明清晰 | 强制规则（见 3.2 节），所有节点 bio 必须事件式简明，禁止学术包装 |

## 8. 验证 / 成功标准

### 8.1 量化（继承 PRD v0.2）

- MVP 8-12 周可 ship
- 访谈 3-5 个目标用户，60% 说「会用、会推荐」
- 上线后第 1 个月内 ≥ 10 个用户主动停留 ≥ 5 分钟

### 8.2 定性（继承 PRD v0.2）

- 1-2 用户主动说「这个独特点是 X，会真的打开来用」
- 1+ 用户主动 push「希望加 Y 功能」（说明产品有粘度）

### 8.3 新增 3 条（基于本次访谈反馈）

- **节点 bio 事件式抽检**：随机抽 10 个节点 bio 给目标用户读，1 分钟内能复述节点核心信息（验证反馈 1 命中）
- **后来者旁注完整度**：12 个核心概念全部带旁注；至少 1 位访谈用户主动评论「后来者旁注是亮点」（验证反馈 8 命中）
- **框架清晰度复述**：第二批访谈给目标用户看 mockup，能否一句话说出"这个产品给的马克思主义发展史框架是什么"（验证反馈 4 命中）

## 9. 远期路线图

### 9.1 V1（2026-Q3，8-12 周）· 本 design 范围

- B+2 双主视图（关系图主 + 地理图画中画副）

### 9.2 V2（V1 上线 3-6 个月后）

- BYOK 对话（Bring Your Own Key 用户自带 LLM 密钥跟节点对话，PRD DR-006）
- **故事模式入口**：首页「故事模式」按钮 → 默认进地理图 tab + 自动播放 Marx 一生地理叙事 + 旁白文字。给路人/学生入口（命中反馈 3 远期愿景）
- 核心概念扩展到 ~20 个（加 资本 / 实践 / 阶级 / 上层建筑细分等）
- 后来者扩展到 ~15 位（加中文马克思学家方向）
- **打开「显示著作」checkbox**（先小规模验证地图节点密度 / 聚合策略 / 重叠处理）

### 9.3 V3+（待定，不预设结论）

- 方案 2 → 方案 1（后来者混入主图）
