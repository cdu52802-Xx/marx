上轮 brainstorming（2026-05-06，PRD v0.2）定到路径 A「学者作品集型」+ 关系星图主视图 + 8-12 周 MVP，然后暂停去做用户访谈校验。2026-05-07 访谈完成，3-5 位目标用户给了 8 条反馈。其中：

## 3. 节点系统

### 3.1 5 类节点 spec

| 类型 | 颜色 | 数量 | 必填字段 (V1) | 选填字段 (V1 录入，V2+ 渲染用) |
|---|---|---|---|---|
| **人** | 紫 | ~50 | `id` / `name_zh` / `name_orig` / `birth_year` / `death_year` / `main_location_lat_lng` / `bio_event_style`(≤5 行) / `citation_urls`[] | `aliases`[] / `portrait_url` / `relation_to_marx` |
| **著作** | 蓝 | ~20 | `id` / `name_zh` / `name_orig` / `pub_year` / `author_id` / `writing_period` / `summary`(≤3 行) / `citation_urls`[] | `version` / `writing_location_lat_lng` ⭐ |
| **事件** | 橙 | ~30 | `id` / `name_zh` / `start_date` / `end_date` / `location_lat_lng` / `description_event_style`(≤3 行) / `participants_ids`[] | `triggered_works_ids`[] |
| **概念** | 绿 | ~15 | `id` / `name_zh` / `name_orig` / `proposed_year` / `proposed_work_id` / `definition_plain`(≤3 行) / `citation_urls`[] / `successor_notes`[] (核心概念) | `related_concepts_ids`[] / `evolution_path` / `first_proposed_location_lat_lng` ⭐ |
| **地点** | 灰 | ~6 | `id` / `name_zh` / `name_orig` / `lat_lng` / `marx_activity_description` | `related_persons_ids`[] / `related_events_ids`[] |

**总计 ~121 节点 + 余量 29 ≈ 150**。

### 3.2 节点 bio 事件式 + 简明强制规则

命中访谈反馈 1（学术语言晦涩冗长）：节点 bio / description / summary 字段**禁止学术包装**，必须采用「x 年 x 月 X 做了什么」事件流写法，每行 ≤ 30 字，总长度 ≤ 5 行（人）/ ≤ 3 行（著作 / 事件 / 概念 / 地点）。

**反例**（学术包装，禁止）：
> 弗里德里希·恩格斯（1820-1895）是 19 世纪伟大的无产阶级革命家、思想家，他与马克思的合作是马克思主义形成发展中的关键因素……

**正例**（事件式简明，必须）：
> 1842年9月 - 在《莱茵报》编辑部第一次见 Marx，关系冷淡
> 1844年8月 - 巴黎再见，10 天彻夜长谈，思想合流
> 1845年 - 合写《神圣家族》《德意志意识形态》
> 1848年 - 合写《共产党宣言》
> 1851-1869年 - 曼彻斯特工厂工作，养活伦敦的 Marx 一家

### 3.3 两图渲染规则

| Tab | V1 默认渲染 | V2+ 可渐进打开 |
|---|---|---|
| **关系图** | 5 类全渲染 (~150 节点) | — |
| **地理图** | 仅紫+橙+灰 三类 (~86 节点) | 「显示著作」checkbox（V2 打开）/「显示概念」checkbox（V3+ 打开） |

**保留这条扩展路径的理由**：用户洞察「历史/地理/政治不分家」——思想形成有强地理上下文。著作的「主要写作地」、概念的「首次提出地」字段在 V1 数据采集时就一并录入，等 V2+ 通过筛选 checkbox 渐进打开。V1 暂不默认渲染是为了控制信息密度风险，**不预设否定**。

## 5. 后来者旁注 panel · 访谈最大变化点

### 5.1 触发位置

**仅在核心概念节点的详情卡里**出现「后来者发展」栏。其他节点（人 / 著作 / 事件 / 地点）的详情卡**不带**这一栏。

### 5.2 V1 必做范围

**12 个核心概念**：
1. 异化 (Verfremdung / Entfremdung)
2. 剩余价值 (Mehrwert)
3. 历史唯物主义 (Historischer Materialismus)
4. 阶级斗争 (Klassenkampf)
5. 商品拜物教 (Warenfetischismus)
6. 意识形态 (Ideologie)
7. 生产方式 (Produktionsweise)
8. 经济基础 / 上层建筑 (Basis / Überbau)
9. 共产主义 (Kommunismus)
10. 革命 (Revolution)
11. 劳动价值论 (Arbeitswerttheorie)
12. 国家 (Staat)

**10 位后来者池**：
卢卡奇 / 葛兰西 / 阿尔都塞 / 哈贝马斯 / 卢森堡 / 列宁 / 马尔库塞 / 哈维 / 詹姆逊 / 巴利巴尔。

每个核心概念挑 3-5 位后来者做旁注（不是 10 位全列）。

### 5.3 旁注内容结构

每条旁注约 300-500 字 + 3-5 个引用源链接。结构示范（以「异化」为例）：

```
[详情卡 · 异化]
─────────────────────────────────
中文：异化      原文：Verfremdung / Entfremdung
提出：1844 年《1844 年经济学哲学手稿》

—— 白话定义 ——
人在资本主义生产中，跟"劳动产品 / 劳动过程 / 自己的本质 / 他人"
四重疏离。例：工人造的车不属于自己。

—— Marx 自己怎么用 ——
1844 手稿首次提出 → 1857-58 大纲改用"对象化" → 1867 资本论隐含
在"商品拜物教"里。

—— 后来者怎么发展 ——  ⭐ 访谈反馈 8 新加
• 卢卡奇 1923《历史与阶级意识》：扩展为"物化"（Verdinglichung），
  推到资本主义意识结构层面
• 阿尔都塞 1965《保卫马克思》：批为"青年 Marx 的人本主义残留"，
  主张用"意识形态国家机器"取代
• 哈贝马斯 1973《合法化危机》：转译为"生活世界殖民化"，
  接入沟通行动理论
• 马尔库塞 1964《单向度的人》：拓展到消费社会的异化新形态

—— 引用源 ——
• MIA《1844 手稿》中/英/德文全文
• SEP "Marx" 词条第 4 节 "Alienation"
• SEP "Lukács" / "Habermas" / "Althusser" 各自词条
```

> **信息密度提示**（v0.3 补 / v1.1 更新）：本节示例展示的是「白话定义 + Marx 自己怎么用 + 后来者怎么发展 + 引用源」四段式完整内容，密度较高。降低密度感的视觉手段（行间距 1.7 / 引用块左侧学者红装饰线 / 章节序号大写罗马数字 / 牛皮纸纹理）已在 § 7 视觉风格定调锁定。是否默认折叠后来者发展 / 引用源 panel 这种交互细节留到 M4 实施时按 mockup A 验证方案敲定。

### 5.4 数据来源

- **主要**：SEP（Stanford Encyclopedia of Philosophy）各后来者词条（卢卡奇 / 哈贝马斯 / 阿尔都塞 / 葛兰西 都有自己的 SEP 词条）
- **二手**：后来者代表作（每位 1-2 本核心著作，如卢卡奇《历史与阶级意识》/ 阿尔都塞《保卫马克思》/ 哈贝马斯《合法化危机》）
- **中文校对**：《剑桥马克思主义指南》中译本 + CNKI 中文学术论文

### 5.5 与原 PRD「不做后续 marxist 思想史扩展」的关系

不冲突。原 PRD 第 5 节排除的是「把后来者做成主图节点」（方案 1）。本 design **V1 范围内**做的是「在概念详情卡里加旁注」（方案 2）—— V1 范围内后来者不出现在主图任一 tab，只在核心概念节点详情卡的下半部分作为参考资料。

**远期不预设否定**：V2+ 根据用户反馈，方案 2 仍可能演化为「后来者混入主图」（方案 1）/「独立 Marx 之后视图」（方案 3）/「方案 2 + 方案 3 双视图切换」。详见 § 10.3（v1.1 章节后移）。PRD v0.3 第 5 节做对应澄清。

## 7. 视觉风格定调（v1.1 新增 · 选定方向 A 学术编辑）

> **状态**：design v1.1 · 视觉风格选型基于 4 风格 mockup 调研后选定方向 A 学术编辑（Editorial Scholar）
> **选型依据**：朋友反馈"A 像查字典，非常适合阅读"——精准命中产品定位「工具型 V1 / 学者优先」（§ 1 / § 10.4）
> **AGENTS.md 合规**：本节回应第 65-69 行硬要求（spec 必含视觉风格定调一节，含 Tone / 字体 / 配色 / 氛围细节）
> **mockup 物证**：[A 学术编辑 mockup](https://cdu52802-xx.github.io/marx/styles/a-editorial.html) · 落选反例 [B 革命粗野](https://cdu52802-xx.github.io/marx/styles/b-brutalism.html) / [C 工业历史](https://cdu52802-xx.github.io/marx/styles/c-industrial.html) / [D 档案文献](https://cdu52802-xx.github.io/marx/styles/d-archive.html)

### 7.1 Tone

**19 世纪学术期刊的现代复刻版**。视觉退到第二位，让 Marx 思想本身（节点 / 概念 / 后来者发展）成为主角。

设计原则（按强度排序）：

1. **不抢戏**：所有装饰服务于内容可读性，不为"记忆点"而设计。学者打开就能用，视觉是透明的。
2. **静谧学术**：refined but not luxe，类似牛津 / 剑桥 / 芝加哥学术出版社的现代版。不是文艺复兴古旧，也不是 IKEA 极简。
3. **典雅严肃**：Marx 思想是"值得郑重对待的文本"。不娱乐化、不刺激。

**不是什么**（反向锚定 4 mockup 调研反馈）：

- ❌ 不是 brutalism / 革命海报 / 巨大 display 字号 / 红黑硬阴影（命中 B 反馈"冲击力太强 / 眼睛累"否定）
- ❌ 不是颗粒纹理 + 钢铁螺帽 + 锈红配色（命中 C 反馈"游戏配色 / 苏丹的游戏 / 同人文学大字典"否定）
- ❌ 不是羊皮纸 + 手写体 + 修改红旁注（命中 D 反馈"海盗 / 迪士尼 / 没有学习氛围"否定）
- ❌ 不是 cyberpunk / 数据可视化炫技（ui-ux-pro-max 脚本初判错误方向）
- ❌ 不是 Web3 / 紫色渐变白底 / 玻璃态等 AI slop

### 7.2 字体方向

| 角色 | 字体 | 备注 |
|---|---|---|
| Display 西文 | **Playfair Display** | 古典对比衬线，h1-h3 / 章节标题 / 装饰 italic |
| Body 西文 | **Source Serif 4** | Adobe 学术专用衬线，长文阅读最佳，opsz 8-60 可变 |
| Body 中文 | **Noto Serif SC**（思源宋体） | 跟 Source Serif 4 同源（Adobe 设计），跨语言一致 |
| Mono / 引文 hash | **JetBrains Mono** | 节点 ID / Wikidata QID / 引用源 hash 用，少量 |

**字号规范**：

| 用途 | 字号 / 行间距 |
|---|---|
| h1 标题 | 32-44px / line-height 1.1 |
| h2 章节 | 22-28px / 1.15 |
| h3 子节 | 16-18px / 1.3 |
| 正文 | 15-16px / **1.7**（学术阅读舒适基线） |
| 节点 label | 12-14px / 1.2 |
| 时间轴 | 13px / 1 |
| 脚注 / meta | 12-13px / small caps + letter-spacing 0.08em |

**避开**（AGENTS.md 第 67 行 + AI slop 反例）：

- ❌ Inter / Roboto / Arial / Space Grotesk / 其他常见 sans-serif
- ❌ 仿宋 / 楷体 / 中文古体（太偏古典书卷，过时观感）
- ❌ 任何无衬线西文 body（破坏学术编辑调性）

### 7.3 配色（CSS 变量）

**主导色板**（全局 import 用）：

```css
:root {
  /* 文本类 */
  --ink: #1A1A1A;          /* 墨黑 - 主文本 / 主节点 / 强对比连线 */
  --ink-soft: #4A4A4A;     /* 软墨 - 次级文本 / meta */

  /* 背景类 */
  --paper: #F4EDE0;        /* 牛皮纸米 - 主背景 */
  --paper-edge: #E8DCCC;   /* 纸边 - 次背景 / hover 底色 */
  --paper-shadow: rgba(26, 26, 26, 0.08);  /* 纸张阴影 */

  /* 规则线 */
  --rule: #D8CCB8;         /* 细分隔线 / 表格边框 */

  /* 强调色（唯一锐利色，仅结构性强调用） */
  --scholar-red: #8B2635;  /* 学者红 - 章节序号 / 引用块左线 / hover / 重要节点 */
}
```

**比例约定**（避 AGENTS.md 第 68 行"温吞均匀调色板"反例）：

- 墨黑 / 文本类：~60% 视觉占比
- 牛皮纸 / 背景类：~30%
- 学者红强调：~10%（仅结构性强调，不撒满）

**反 AI slop 检查**：

| 反例 | A 路线对应 |
|---|---|
| 紫色渐变 + 白底 | 牛皮纸米 + 墨黑（命中"主导色 + 锐利强调色"模式） |
| 均匀 5 色调色板 | 60/30/10 锐利比例 |
| 玻璃态 / 模糊渐变 | 纸张噪点 + 实色 |

## 9. 验证 / 成功标准

### 8.1 量化（继承 PRD v0.2）

- MVP 8-12 周可 ship
- 访谈 3-5 个目标用户，60% 说「会用、会推荐」
- 上线后第 1 个月内 ≥ 10 个用户主动停留 ≥ 5 分钟

### 8.2 定性（继承 PRD v0.2）

- 1-2 用户主动说「这个独特点是 X，会真的打开来用」
- 1+ 用户主动 push「希望加 Y 功能」（说明产品有粘度）

### 8.3 新增 3 条（基于本次访谈反馈）

- **节点 bio 事件式抽检**：随机抽 10 个节点 bio 给目标用户读，1 分钟内能复述节点核心信息（验证反馈 1 命中）
- **后来者旁注完整度**：12 个核心概念全部带旁注；至少 1 位访谈用户主动评论「后来者旁注是亮点」（验证反馈 8 命中）
- **框架清晰度复述**：第二批访谈给目标用户看 mockup，能否一句话说出"这个产品给的马克思主义发展史框架是什么"（验证反馈 4 命中）
