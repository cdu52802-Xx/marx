## 3. Lessons Learned（5-8 条 / 跨 milestone 让 M5 更易做）

### Lesson 1 · vision pivot 决策成本高 / 但延迟更高
M3 → M4 vision pivot 触发 = PM Task 10 demo review 反馈"跟 M2 视觉无区别 / 信息密度不够"。如果 PM 不及时反馈 / M4 继续按 M3 范围跑完 = 50 节点 person-network ship 后浪费 / 再回头重做成本更大。

**M5 借鉴**：每个主线 implementation 进行到 ~30% 时强制 PM checkpoint（不是 100% 完成才 review）/ vision-level 偏差立即拍 / polish 攒到 backlog 批量处理（M4 T6 PM 3 轮 checkpoint 通过 = 这个套路 work）。

### Lesson 2 · hybrid 数据模式 work（AI 草稿 + PM 异步复核）
M4 T2-T5 跑 hybrid（commit chain 8 轮 / 92 ClaimNode + 31 ClaimRelation 入库不阻塞）+ PM 任意时间打开 checklist md 复核。PM 原话 "**别让人工审核成为进度卡点**"。M5 主线 C 多类型详情页采集（person bio / work 简介）继续 hybrid。

### Lesson 3 · subagent task 拆太碎反而慢
M4 plan 13 task / T0-T9 部分被 subagent-driven 模式跑（5 轮 implementer + reviewer 来回）/ 每 task 切换 context 开销不低。M5 建议**单线启动**（A 或 B 或 C 选一 / 跑完再启第二条）/ 每线 task 拆 5-7 个 / 不拆 13 个。

### Lesson 4 · 第三方参考的真实角色（denizcemonduygu 校准 memory）
T5 SOFT-BLOCK 后 PM 校准：denizcemonduygu = **视觉/布局参考为主 + 数据副产品为辅 / 不是项目"标准答案"**。M5 主线 B（philosophy_vis 朋友项目）同样 framing = 借右侧栏滑入交互范式 / 不是抄实现。落 memory feedback_third_party_reference_role.md。

### Lesson 5 · 视觉美学三件套（AGENTS.md）实战 work
M4 brainstorm + spec + implementation 三件套硬约束 = 避免了 generic AI slop。spec § 4 视觉风格定调（学术编辑风 + EB Garamond + 紫主导）从 brainstorm 一直贯穿到 T9 详情栏 hybrid 配色。M5 继续遵守 / 万能打底提示词内置。

### Lesson 6 · PRD trace 工作流改进 A+B+C 落地补丁
M4 vision pivot 时 silent drift 地理图副视图 = 暴露 PRD 跟 implementation 脱节问题。2026-05-12 PM approve A+B+C：
- A · takeaway 必含 PRD 覆盖率（本节 § 1 已落）
- B · 新 milestone spec 必含 PRD trace 节
- C · PRD V1 必做加实施状态注释

**M5 spec 启动时 PRD trace 节是强约束** = 防止 M5 再次 silent drift。

### Lesson 7 · polish 节奏（vision-level 立即修 / polish 攒 backlog）
M4 T9 PM 反馈右侧栏改造 = vision-level 立即修（commit 8866810）/ 详情页 4 维度细节 PM 暂无方向 = 落 polish backlog 慢磨。**避免 polish 死循环**（每 task 完都死磕细节 / 占用主线时间）。M5 继续这套节奏。

### Lesson 8 · M5 上线前必做 = archive M4 demo（防 silent drift）
**M5 zoom + 多详情页 + header = M4 demo 不可逆覆盖**。M5 上线前必做（前置 task）：
```bash
npm run build
mkdir -p public/m4-archive
cp -r dist/* public/m4-archive/
git tag m5-pre-archive
```
