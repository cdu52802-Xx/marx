## 1. 一句话定义（Elevator Pitch）

> 这是一个给 **哲学/政治经济学/社科 研究者**（次要：进阶读者）用 **以 Marx 为中心、人/著作/事件/概念/地点 多类异质节点的关系星图** 解决 **「Marx 思想形成路径难以可视化」** 这一痛点的 **学者作品集型交互可视化产品**。

---

## 2. 背景与动机

### 灵感来源
朋友的 [philosophy_vis](https://github.com/Sia12345678/philosophy_vis) 项目——把 118 位东西方哲学家做成全景星图（节点同质、节点都是哲学家）。受其启发但走另一条路：**单中心 + 异质节点 + 思想形成叙事**。

### 已有方案的不足
- **MIA / Marxists.org**：内容广度天花板，但 1990s 风格，没有可视化
- **SEP（斯坦福哲学百科）Karl Marx 词条**：内容深度天花板，但是单页文章，没有可视化
- **Wikipedia / Britannica**：传统百科页面，关系靠 hyperlink，没有图谱
- **Dickinson Marx 数字历史项目（2017）**：跟我们最像，但只做了时间线 + Google Earth，**没有思想/概念/关系图**
- **Önduygu 哲学史可视化**：策展深度天花板，但是哲学史全景，不是单人 Marx 深度
- **中文 Marx 可视化**：基本空白

### 切入点
「以 Marx 为中心 + 多类异质节点 + 思想形成叙事」这个细分赛道几乎空白，**中文学者市场是结构性蓝海**。

---

## 3. 目标用户

### 主要用户画像
- **角色**：哲学 / 政治经济学 / 社科 研究者（在读硕博、青年学者）
- **场景**：研究、教学、写作时需要快速定位 Marx 生命中的人际、著作、概念、事件之间的关系
- **痛点**：现有资源把人、思想、时代分开讲，难以「看见」思想形成路径

### 次要用户画像
- 中文 Marx 研究 / 左翼内容创作者（公众号、B 站 up 主）
- 进阶读者（读过《资本论》通俗版或《1844 手稿》，想看清思想形成路径）

### 非目标用户（明确不服务）
- 纯入门读者（需要强引导、低门槛、概念名词解释、推荐路线 这种 dumb-down 功能的）
- 反对学术严肃度的「快速速成型」用户

---

## 4. 核心功能（MVP 范围）

### 已确认 V1 必做
- [x] **关系星图主视图**：Marx 居中 + 多类异质节点辐射（人 / 著作 / 事件 / 概念 / 地点）
- [x] **时间轴副视图**：1818-1883，可拖动滑块按生卒/出版年份淡入淡出
- [x] **地理足迹副视图**：特里尔 → 波恩 → 柏林 → 巴黎 → 布鲁塞尔 → 伦敦
- [x] **节点点击侧边栏**：中文 bio + 完整字段 + 学术引用源链接
- [x] **关系连线 6-8 种类型**：师承 / 论敌 / 朋友 / 影响 / 作者 / 提出概念 / 居住 / 参与事件，**视觉区分**（颜色 / 粗细 / 虚实）
- [x] **悬浮 tooltip**：节点姓名（中 + 原文）+ 1 行核心信息
- [x] **筛选 / 搜索**：按节点类型、按关系类型、按人名（中文 + 原文）
- [x] **缩放 + 平移**：1×–8× 缩放，放大查看密集子图

### 数据规模
~150 节点 = ~50 人 + ~20 著作 + ~15 概念 + ~30 关键事件

### 数据严肃度
**学术级**——每节点必须带可信引用源：
- **MIA**（Marxists Internet Archive，马克思恩格斯互联网档案）—— 著作原文 + 时间线
- **SEP**（Stanford Encyclopedia of Philosophy，斯坦福哲学百科）—— 概念定义
- **MEGA**（Marx-Engels-Gesamtausgabe，马恩全集历史考证版）—— 学术引文标准
- **Wikidata**（SPARQL 拉关系骨架：P737 受影响于 / P800 著名作品 / P1066 老师 / P802 学生）

### 语言策略
中文为主，原文德/英术语 + 中文翻译注释（举例：Verfremdung（异化）/ Mehrwert（剩余价值））

### 交付形态
**纯静态网页** —— Vite + TypeScript + D3.js + 原生 DOM（跟 philosophy_vis 同栈，可借朋友经验）。`vite build` 产出 `dist/`，部署到 GitHub Pages 或 Cloudflare Pages，零运维。

## 5. 不做的事（V1 明确边界）

- **BYOK 跟 Marx 对话**（Bring Your Own Key 用户自带 LLM 密钥）—— V2 加，V1 不做（先跑通主线）
- **后续 marxist 思想史扩展**（列宁 / 卢森堡 / 葛兰西 / 阿尔都塞 / 哈维...）—— 范围炸裂风险
- **多语言**（中英德三语）—— 中文 + 关键术语原文已够，全双语工作量翻倍
- **用户笔记 / 协同标注 / 后端账号系统** —— 静态产品形态不要后端
- **入门引导路线 / 名词解释这种 dumb-down 功能** —— 不服务入门用户
- **时代背景节点**（工业革命 / 1848 革命 / 巴黎公社 作为节点）—— 范围 C 选项，工作量翻倍，**V1 不做但相关事件作为 Marx 参与事件节点保留**

---

## 6. 成功标准

### 量化指标
- MVP 8-12 周可 ship
- 访谈 3-5 个目标用户，至少 60% 说「会用、会推荐」
- 上线后第 1 个月内：≥ 10 个用户主动停留 ≥ 5 分钟（学者用户口碑传播门槛）

### 定性标准
- 至少 1-2 个目标用户访谈对象**主动**说「这个独特点是 X，我会真的打开来用」
- 至少 1 个上线试用者主动**反向 push**：「我希望加 Y 功能」（说明产品有粘度）

---

## 7. 风险与未知

### 已知风险
- **R1：路径 A 是否真有学者价值** —— 待用户访谈校验（节点 ①）
- **R2：一人公司 scope 失控** —— 用 ~150 节点 + 8-12 周 MVP 控制
- **R3：LLM 时代「内容平台红海化」** —— 护城河必须是策展质量 + 中文学者社区认可，不能凑合数据
- **R4：Önduygu 风格被人扩展到 Marx 单人** —— 我们要在中文市场 + 单中心叙事上抢身位

### 待验证假设
- 学者用户**真的会**从「人/书/事件/概念混合关系图」切入研究 Marx——这是核心假设，访谈必校验
- **付费意愿**——一人公司商业化路径，访谈题 6 校验
