## 哪些坑值得记下来（M2+ 避免重踩）

### 坑 1 · GitHub Pages 不支持免费私仓 ⭐ 最大坑

**症状**：Task 7 Step 2 让用户去 GitHub Settings > Pages 改 Source = GitHub Actions，但页面显示 "Upgrade or make this repository public to enable Pages" —— 没有 "Build and deployment" 节。

**原因**：GitHub 免费个人账户的 private repo **不支持 Pages**。必须 public 或升级 Pro（4 美元/月）或换 Vercel/Cloudflare Pages。

**Marx 的处理**：选 (a) 改 public —— V1 PRD 本来就是公开产品。改之前先扫了 docs/ 没敏感信息（无 email / 手机号 / token / 访谈对象 PII）。

**M2+ 防御**：plan 顶部"0 代码 PM 准备工作"应加一节「**部署平台前置检查**」：
- 确认 marx repo 是 public（私仓 → 升 Pro / 改 public / 换 Vercel 三选一）
- 这条放在 Task 1 之前提醒，避免 Task 7 才发现

### 坑 2 · Prettier 默认会重排 markdown

**症状**：Task 6 Step 5 跑 `npm run format`，prettier 按 `printWidth: 100` 把 8 个 markdown 文档（PRD / design / takeaway / plan / AGENTS / README）全重排，227+/188- 行。

**原因**：`prettier --write .` 对 *.md 默认生效，不区分代码 vs 手写文档。

**Marx 的处理**：`git restore` 还原 markdown + `.prettierignore` 加 `*.md` 显式排除。

**M2+ 防御**：项目模板里的 `.prettierignore` 一次到位含 `*.md`。

### 坑 3 · Node.js 20 GitHub Actions 退役预警

**症状**：CI workflow 跑通但带 deprecation 警告 —— `actions/checkout@v4` 等基于 Node.js 20，**2026-09-16 后会被 GitHub 强制移除**。

**当前状态**：不阻塞 M1（4 个月缓冲）。

**M2+ 防御**：M3-M4 期间监控 `actions/checkout` `actions/setup-node` `actions/upload-pages-artifact` `actions/deploy-pages` 是否有支持 Node.js 24 的新版本，及时升级。

### 坑 4 · plan 写的版本号是下限，npm 装的是最新

**症状**：plan Tech Stack 写 Vite ^5.0 / TS ^5.4，npm install 实际装了 Vite 8 / TS 6。

**当前状态**：M1 范围 API 兼容，没遇到问题。

**M2+ 防御**：写 plan 时如果有版本兼容硬要求，写明确版本（如 `vite@5.4.x`）；否则下限即可。

### 坑 5 · 本地 Node v24 vs CI Node v20

**当前状态**：API 兼容，无冲突。

**M2+ 防御**：如果 M2/M3 出现 "本地跑通 CI 失败" 的诡异情况，第一直觉先看 Node 版本差异。
